# 仓库协作规范

- 禁止使用GPT5.4以下、或Claude Code Opus4.7以下模型进行AI辅助开发，所有模型我都已测试过，目前只有GPT5.5和Claucde Code Opus4.8能够胜任开发，但是在国内你很难蹬到真正的Opus 4.8，所以本项目开发统一使用GPT5.5，且必须是Pro里的GPT5.5，Plus里的会降智。
- 在你提交PR之前，如果对dll测公共代码库进行了修改，例如DirectXTK升级，则需要同时测试DX11部分和DX12部分，确保无兼容性问题，不要没测试就提交上来。
- 不要提交负优化的结果，如果你的代码修改之后让某个机制更加卡顿了，就不要提交上来。
- 任何情况下，只要你修改了dll代码，必须进行测试，不测试不允许提交上来。
- 代码中不允许出现任何中文注释!!!只有git提交描述以及文档可以用中文写，其它的都不许用中文防止出现乱码，AI遇到乱码会炸。
- 每次pull request只允许修改一个特性，超过一个特性修改的pull request将会被拒绝，因为无法在短时间内进行复现测试，除非你能拿出有力的证据证明你开发的结果是对的，例如附带完整测试视频等等。
- 所有PR我都会去人工测试复现结果，以及使用GPT5.5审查PR内容，如果有问题将打回重做。
- 在你要提交PR之前，请在自己的Fork版本中进行充分的测试，非必要不提交回来，有条件请在自己的fork版本中开发，然后将你的成果总结成文档提交到issue中详细说明即可，PR的合并是很危险的。

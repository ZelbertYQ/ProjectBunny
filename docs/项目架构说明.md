# 项目架构说明

zaychik目录下的Blender插件，可以在VSCode里Ctrl+Shift+P启动Blender调试来使用

仅作为初期部分数据分析使用，可参考其解析方式，实现自己的模型提取脚本

模型提取目前已经在SSMT4中实装部分测试，后期将放到开源项目中，你可以先拿SSMT4进行模型提取测试

或者实现自己的模型提取到导入全套流程。

# 为什么我认为DX12其实很简单

1.DX11部分打下了良好的基础，大部分内容可以复用
2.DX12的Mod替换本质上是架构问题，因为现在有AI的情况下，编码不需要人力了（我大部分代码都是躺着写完的，没错，把电竞椅展开躺那儿，然后偶尔敲敲键盘动动鼠标指挥AI去写的，大部分时候我在听音乐玩手机，我负责的内容就是确保架构正确，以及充当人力测试）
3.DX12文档非常规范，且数十年没有变更，那么理论上这个东西是存在的，只是需要有人去指挥AI把结果不断收敛到确定性的那个时空。
